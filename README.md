# 网页监控服务 (Monitor Dashboard)

这是一个用于监控指定网页内容变更（如公告、新闻）并发送企业微信通知的 Go 语言服务。包含一个现代化的 Web 管理后台，支持多目标监控、可视化配置、历史记录查询及手动重推通知。

## 功能特性

- **多目标监控**: 支持同时监控多个网页 URL。
- **现代化 Web 后台**:
    - **仪表盘**: 实时查看监控状态和运行日志。
    - **历史记录**: 查看已抓取的公告历史，支持**排序**、**来源筛选**及**分页**（10/20/50/100条）。
    - **手动重推**: 支持对任意历史记录进行**手动推送**，方便补发通知。
    - **配置管理**: 在线添加、编辑、删除监控目标，支持关键字过滤。
    - **一键启停**: 可以在界面上快速启用或停用某个监控目标，且启用时会**立即进行一次检查**。
- **通用配置**:
    - 在线修改 **Webhook URL** 和 **检查间隔**。
    - 修改检查间隔后**立即生效**，无需重启服务。
- **持久化存储**: 使用 SQLite 数据库 (`monitor.db`) 存储配置和历史记录，数据更安全。
- **智能去重**: 自动防止重复发送通知。

## 快速开始

### 1. 编译和运行

确保已安装 Go 1.21+ 和 Chrome 浏览器。

```bash
# 编译
go build -o monitor main.go

# 运行服务 (默认监听 8080 端口)
./monitor
```

### 2. 访问管理后台

浏览器打开: [http://localhost:8080](http://localhost:8080)

### 3. 配置通知

在 "通用配置" (Generic Config) 标签页中，设置你的企业微信 Webhook URL。
例如: `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxx`

### 4. 添加监控目标

在 "监控配置" (Monitor Config) 标签页中，点击 "+ 添加目标"。
- **名称**: 目标名称（如：通知公告）
- **URL**: 要监控的网页地址
- **关键字**: (可选) 仅通知包含特定关键字的公告，多个关键字用逗号分隔。
- **启用状态**: 默认启用。

## 功能截图

- **通知历史**: 支持按发布时间/抓取时间排序，按来源筛选，分页显示。
- **手动推送**: 历史列表右侧 "操作" 列包含 "推送" 按钮。
- **目标管理**: 卡片式展示监控目标，支持快速开关。

## 文件说明

- `main.go`: 后端核心代码 (HTTP Server, Crawler, SQLite Manager)。
- `monitor.db`: SQLite 数据库文件 (自动生成)。
- `config.json`: 初始配置文件 (首次运行时用于导入数据，后续配置由 DB 接管)。
- `static/`: 前端静态资源 (HTML, CSS, JS)。

## 技术栈

- **后端**: Go, SQLite (modernc.org/sqlite), Chromedp (Headless Chrome), Goquery
- **前端**: HTML5, Vanilla JavaScript, CSS3
- **通信**: JSON API

## 注意事项

- 程序依赖 Chrome 浏览器进行页面渲染（解决单纯 HTML 抓取无法获取动态内容的问题）。
- 首次运行时，如果没有 `monitor.db`，会自动初始化并尝试从 `config.json` 导入配置。
- 修改检查间隔后，下一次检查会根据新的间隔立即重置计时器。
